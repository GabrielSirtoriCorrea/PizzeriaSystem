plataformas e linguagens = Android Studio (App), Python(Server), Java (Desktop), Banco de dados SQLite

App:
	Objetivo: Enviar pedidos

	Mostrar na tela: Endereço de entrega, promoções, informações da pizzaria, area de novo pedido

	enviar json: {
                                    ID: "",
		Cliente: "",
		Telefone: "",
	                  endereço: "",
		Tipo de pizza: "",
		Borda: "",
		Qntd sabores: 1,
		Sabor1: "",
		Sabor2: "",
		Observações: "",
		Valor total: "",
		
		}


	Tarefas:
	
		Usuário escolhe o pedido
		Envia o pedido para o servidor
		Espera resposta de confirmação do Servidor


                 Classes: Adapter (Converte as pizzas para serem exibidas no ListView)
                              SaborPizza (Dá origem a todas as pizzas disponiveis )
                              Pedido (Dá origem a todas as caracteristicas do pedido a serem enviadas pelo JSON)]
                              Adreess (Classe da activity Address)
                              Request (Classe da activity Request)
                              SecurityPreferences(Armazena: Cliente, telefone, endereço)
	            SocketConnection (Cria comunicação com o servidor)


                  Atributos SaborPizza: Sabor
                                                    preço

	Métodos SaborPizza: Constructor, Getters e setters
                  
	Atributos Pedido: Cliente, Telefone, Endereço, Tipo de pizza, Borda, Qntd sabores, PizzaSabor1, PizzaSabor2, observações, valorTotal

	MétodosPedido: Constructor, Geters, setters e PrepararPedido

		PrepararPedido(): Pega todos os atributos da classe e converte para um JSON  

                  *ValorTotal definido pelo Valor do PizzaSabor1 + PizzaSabor2 
                                          
		

Server:
	Objetivo: Injetar dados recebidos no banco de dados
	
	Tarefa1 (Thread com o Usuário):
		Receber pedido JSON do usuário
		Armazenar no banco de dados com o estado EM ANDAMENTO
		Enviar resposta para o Usuário

	Tarefa2 (Thread com o Desktop):
		Enviar continuamente os dados do banco de dados
		Receber requisição de pedido completo e mudar estado do pedido para COMPLETO

Banco de dados:

	Tabela: Pedido
	Colunas: Cliente, Telefone, endereço, tipo da pizza, borda, qntd de sabores, sabor1, sabor2, Observações, Valor total, estado Pedido
 	

Desktop:
	Classes: SocketConnection(Cria comunicação com o servidor)
                               Pedido

	Aba pedidos atuais:
		Mostra todos os pedidos do banco de dados com o estado EM ANDAMENTO ao lado(Lista de espera)
		Mostra no centro da tela os pedidos que estão sendo feitos no momento
		
	Aba Histórico de pedidos:
		Mostra todos os pedidos do banco de dados
		
	Aba Estatisticas:
		Mostra um gráfico com os sabores mais pedidos
		Mostra a renda total
